Factory Module

The Factory module in ProDash Tools automates the creation of components and configurations using predefined patterns and templates.

Core Concepts

  • Templates - Reusable patterns for generating new components
  • Plugins - Extend factory functionality for specific frameworks
  • Projects - Encapsulated development workspaces with factory integration

Example: Component Factory

const factory = new ComponentFactory();
const button = factory.create('button', {
  variant: 'primary',
  label: 'Click Me'
});

Context Keeper

The Context Keeper is a robust system for maintaining and managing chat contexts, backups, and system health monitoring. It runs directly in your ProDash_Tools root directory.

Key Features

  • Context Management - Store and retrieve conversation contexts with rich metadata
  • Automatic Backups - Scheduled backups every 60 minutes with manual backup option
  • Health Monitoring - System health checks every 15 minutes
  • Git Integration - Automatic repository detection and integration
  • Context Capture - Automatic context capture every 5 minutes
  • Dynamic Port Management - Consistent port assignment based on project name

Using Context Keeper

Context Keeper tools can be accessed through both the web dashboard and command-line utilities:

Dashboard Access

# Start the backend server
npm start

# In a new terminal, start the frontend
cd dashboard/public && npm run dev

The dashboard will be available at the dynamically assigned port (53xxx range), and the API will be accessible at its corresponding port (54xxx range).

Command Line Access

After adding the command wrapper to your shell profile:

source ~/Projects/ProDash_Tools/cli/command_wrapper.sh

Available commands:

# List all contexts
context-tools list

# Get a specific context
context-tools get [id]

# Create a manual backup
context-tools backup

# Restore from a backup
context-tools restore [filename]

# Check system health
context-tools health

Port Configuration

Context Keeper uses consistent port ranges based on your project name hash:

  • Frontend services: 53xxx range (calculated from project name)
  • Backend services: 54xxx range (calculated from project name)
  • Database services: 55xxx range (calculated from project name)

For example, if your project hash is 420:

Frontend: 53420
Backend:  54420
Database: 55420

This ensures consistent port assignments across restarts and minimizes conflicts between different projects.

Automatic Features

  • Backups: Created automatically every 60 minutes
  • Health Checks: Run every 15 minutes
  • Context Capture: Performed every 5 minutes
  • Git Detection: Automatic on startup

Creating New Projects

To create a new project with Context Keeper integration:

prodash-create my-new-project

This will:

  • Create a new project directory
  • Set up the necessary file structure
  • Configure dynamic port management
  • Link core Context Keeper utilities
  • Initialize Git repository (if needed)

Builder Module

The Builder module provides a fluent API for constructing complex components and configurations with step-by-step object creation.

Key Features

  • Supports complex object construction
  • Provides method chaining
  • Enforces construction rules

Example: Component Builder

const component = new ComponentBuilder()
  .setName('MyComponent')
  .setType('container')
  .addChild(childComponent)
  .build();

Database Configuration

ProDash Tools supports multiple database types with flexible configuration options.

Supported Databases

  • MySQL
  • PostgreSQL
  • SQLite

Configuration Example

{
  "type": "postgresql",
  "host": "localhost",
  "port": 5432,
  "username": "postgres",
  "database": "prodash_db"
}
