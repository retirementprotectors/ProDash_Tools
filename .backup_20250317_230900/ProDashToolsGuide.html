Tools Guide

Factory Module

The Factory module in ProDash Tools automates the creation of components and configurations using predefined patterns and templates.

Core Concepts

  • Templates - Reusable patterns for generating new components
  • Plugins - Extend factory functionality for specific frameworks
  • Projects - Encapsulated development workspaces with factory integration

Example: Component Factory

const factory = new ComponentFactory();
const button = factory.create('button', {
  variant: 'primary',
  label: 'Click Me'
});

Context Keeper

The Context Keeper is a robust system for maintaining and managing chat contexts, backups, and system health monitoring.

Key Features

  • Context Management - Store and retrieve conversation contexts with rich metadata
  • Automatic Backups - Schedule regular backups of all contexts
  • Health Monitoring - Track system metrics and receive alerts for issues
  • Dashboard - Visual interface for managing the system
  • Dynamic Port Configuration - Smart port assignment to avoid conflicts

Using Context Keeper

Context Keeper tools can be accessed through both the web dashboard and command-line utilities:

Dashboard Access

npm run dev

The dashboard will be available at the dynamically assigned port (typically in the 51xxx range).

Command Line Access

After adding the command wrapper to your shell profile:

source ~/Projects/ProDash_Tools/cli/command_wrapper.sh

You can use these commands:

# List all contexts
context-tools list

# Get a specific context
context-tools get [id]

# Create a backup
context-tools backup

# Restore from a backup
context-tools restore [filename]

# Check system health
context-tools health

Creating Projects with Context Keeper

New projects can automatically integrate with Context Keeper using the prodash-create command:

prodash-create my-new-project

This creates a new project with Context Keeper already linked, providing context-awareness and monitoring.

Dynamic Port Management

Context Keeper uses port ranges based on project name:

  • Frontend services: 53xxx range
  • Backend services: 54xxx range
  • Database services: 55xxx range

This ensures consistent port assignments and minimizes port conflicts.

Builder Module

The Builder module provides a fluent API for constructing complex components and configurations with step-by-step object creation.

Key Features

  • Supports complex object construction
  • Provides method chaining
  • Enforces construction rules

Example: Component Builder

const component = new ComponentBuilder()
  .setName('MyComponent')
  .setType('container')
  .addChild(childComponent)
  .build();

Database Configuration

ProDash Tools supports multiple database types with flexible configuration options.

Supported Databases

  • MySQL
  • PostgreSQL
  • SQLite

Configuration Example

{
  "type": "postgresql",
  "host": "localhost",
  "port": 5432,
  "username": "postgres",
  "database": "prodash_db"
}
